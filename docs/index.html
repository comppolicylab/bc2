Bookkeeping

Up 
post /exposure
Log an exposure event (logExposure)
Posting an "exposure" event is required whenever someone views a case. This must happen regardless whether the case is blinded or not. We need to see who is viewing cases and when in order to monitor and improve our application.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

Exposure Exposure (required)
Body Parameter —

Responses

201

Accepted

Posting an "outcome" event is required whenever someone makes a decision on a case. This must happen regardless whether the case is blinded or not. We need to see who is making decisions and when in order to monitor and improve our application.

Up 
get /redact/{jurisdictionId}/{caseId}
Get status of document redaction for a case. (getRedactionStatus)
Get the status of redaction for all documents in a case. This will return a list of document IDs and their redaction status.

Path parameters

jurisdictionId (required)
Path Parameter — The jurisdiction ID default: null
caseId (required)
Path Parameter — The case ID default: null

Return type

RedactionStatus

Example data

Content-Type: application/json
{
  "documents" : [ {
    "jurisdictionId" : "jurisdictionId",
    "caseId" : "caseId",
    "documentUrl" : "https://openapi-generator.tech",
    "documentId" : "documentId",
    "status" : "COMPLETE"
  }, {
    "jurisdictionId" : "jurisdictionId",
    "caseId" : "caseId",
    "documentUrl" : "https://openapi-generator.tech",
    "documentId" : "documentId",
    "status" : "COMPLETE"
  } ],
  "jurisdictionId" : "jurisdictionId",
  "caseId" : "caseId"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

OK RedactionStatus

post /redact
Redact a document (redactDocument)
Submit a document for redaction. Redaction happens asynchronously and may take some time. When finished, the redacted document will be posted to the provided callback URL.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json
